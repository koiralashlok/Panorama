2: FAST feature detector

2: FASTR feature detector

For the first image (the cereal box), almost no corners were removed by the Harris cornerness measure. This is because the detected corners are useful corners and changing the threshold further lost important information For the lamp on the other hand, there are very few features and having a higher threshold would lose part of limited information in the image causing issues for the panorama step The average time taken to find the FAST features was 0.2381s and for FASTR features after fast features were already computed was 0.2588s, resulting in a cumulative time of roughly 0.5s Although this is not a lot, it is mathematically double of simply finding FAST features. However, this is inexpensive compared to the time save in later stages

3: Point description and matching







Used FREAK for cereal box (first set) and ORB for the remaining For the first set, points that were matched using the FASTR features have many more matching corners compared to the FAST points In the second set however, only 1 more match is found using the FASTR points versus the FAST points since it has less features to begin with. The FAST computation took 0.2005s while the FASTR computation took 0.1008s which is only half the time as the former or twice as fast!

4: RANSAC and Panoramas







The maximum number of iterations was set to 100 for the FASTR feature panoramas. This resulted in really good looking and well matched panoramas. On the other hand, the maximum number of trials was set to 10000 for the FAST feature panoramas which resulted in mostly good images. The FASTR maximum number of trials (100) is much is much lesser than that for the FAST (10000).
